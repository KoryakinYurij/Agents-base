# Руководство для ИИ-агентов по работе с этим репозиторием

## Основная информация

*   **Цель:** Разработка, тестирование и хранение ИИ-агентов.
*   **Основная технология:** Google Agent Development Kit (ADK) на языке Python.
*   **Модель:** По умолчанию используется `gemini-1.5-flash`, но можно настроить и другие модели.

## Рабочий процесс

### 1. Создание нового агента

1.  **Используйте команду `adk create`:** Для создания нового агента всегда используйте стандартную команду ADK.
    ```bash
    adk create src/имя_нового_агента
    ```
2.  **Расположение:** Все агенты должны находиться в директории `src/`.
3.  **Конфигурация:** В файле `agent.py` нового агента обязательно заполните следующие поля:
    *   `name`: Уникальное имя агента.
    *   `description`: Краткое и понятное описание того, что делает агент.
    *   `instruction`: Четкая инструкция для LLM о том, как себя вести и какие инструменты использовать.
4.  **Инструменты:** Если вы создаете новые инструменты (функции), убедитесь, что у них есть подробные docstrings, объясняющие их назначение, аргументы и возвращаемое значение.

### 2. Тестирование агента

1.  **Цель тестирования:** Убедиться, что агент корректно выполняет свою задачу, описанную в `description`.
2.  **Методы тестирования:**
    *   **Интерактивное тестирование в консоли:**
        ```bash
        adk run src/имя_агента
        ```
    *   **Тестирование через веб-интерфейс:**
        ```bash
        adk web --port 8000
        ```
        (Запускать из корневой директории проекта)
3.  **Отчет об ошибках:** Если агент не справляется с задачей, подробно задокументируйте проблему, входные данные и полученный результат. Если найдено решение, занесите его в `TROUBLESHOOTING.md`.

### 3. Работа с документацией

*   **`README.md`:** Если в процессе работы изменяются инструкции по установке или запуску проекта, обновите этот файл.
*   **`TROUBLESHOOTING.md`:** При столкновении с неочевидной проблемой, которая требует времени на решение, обязательно задокументируйте проблему и ее решение в этом файле. Особенно это касается проблем, связанных с версиями библиотек или устаревшей документацией.

### 4. Управление зависимостями и API-ключами

*   **Зависимости:** Если для работы агента требуются новые Python-библиотеки, их следует добавить в файл `requirements.txt` (если он есть) и убедиться, что они устанавливаются корректно.
*   **Ключи API:** **Никогда не добавляйте ключи API напрямую в код.** Используйте для этого файл `.env`, который находится в `.gitignore`. Шаблон для этого файла — `.env.example`.
